# 🔊 音频增强功能说明

## 🎯 问题和解决方案

### 问题：测试音能听到，但TTS音频听不到

**原因**：TTS音频振幅太小

- 测试音振幅：0.3（手动设置）
- TTS音频振幅：0.05 - 0.15（由TTS服务生成）
- 会议软件可能有音量阈值，过滤了小信号

### 解决方案：音频增强

自动检测和调整音频振幅到最佳水平。

---

## ✅ 已实施的增强功能

### 1. 音频标准化

```python
target_amplitude = 0.6  # 目标振幅
gain = target_amplitude / original_max  # 计算增益
audio_float = audio_float * gain  # 应用增益
```

**效果**：
- 自动将音频放大到合适的音量
- 确保会议软件能听到

### 2. 智能增益限制

```python
if gain > 5.0:  # 最多放大5倍
    gain = 5.0
elif gain < 0.5:  # 最少0.5倍
    gain = 0.5
```

**效果**：
- 避免过度放大噪音
- 避免过度衰减正常音频

### 3. 软削波处理

```python
audio_float = np.tanh(audio_float * 1.2) * 0.9
```

**效果**：
- 避免音频削波失真
- 使用 tanh 函数平滑地限制振幅
- 比硬削波音质更好

### 4. 改进的重采样

```python
# 使用 scipy.signal.resample（高质量）
audio_float = signal.resample(audio_float, target_length)
```

**效果**：
- 更好的音质
- 减少重采样导致的失真
- 如果没有 scipy，自动 fallback 到线性插值

## 📊 音频处理流程

```
1. 接收TTS音频（PCM 16-bit, 24000Hz）
   ↓
2. 转换为float32（-1.0 到 1.0）
   ↓
3. 检查原始振幅
   原始最大振幅：0.123
   原始RMS能量：0.045
   ↓
4. 重采样（24000Hz → 48000Hz）
   使用高质量算法
   ↓
5. 计算增益
   目标振幅：0.6
   增益：0.6 / 0.123 = 4.88x
   ↓
6. 应用增益
   audio *= 4.88
   ↓
7. 软削波
   避免削波失真
   ↓
8. 输出到CABLE Input
   最终最大振幅：0.600
   最终RMS能量：0.220
   ↓
9. 传输到会议软件 ✅
```

### 问题2：音质不好/有失真

**可能原因**：
- 增益过大导致失真
- 软削波参数不合适

**解决**：
```python
# 降低目标振幅
target_amplitude = 0.5  # 从0.6改为0.5

# 或者调整软削波

```

### 问题3：音量不稳定

**可能原因**：
- TTS服务返回的音频振幅波动大
- 增益计算基于最大振幅（不是RMS）

**解决**：
可以改用RMS标准化（更稳定）：

```python
# 替换第340行
# 原来：基于最大振幅
gain = target_amplitude / original_max

# 改为：基于RMS
target_rms = 0.2  # 目标RMS
gain = target_rms / original_rms
```

## 📖 技术细节

### 为什么使用 tanh 软削波？

**tanh函数特性**：
- 输入小时，输出≈输入（线性）
- 输入大时，输出平滑地趋向±1
- 避免硬削波的突变失真

**对比**：
```
硬削波：
if x > 1.0:
    x = 1.0
→ 突变，产生谐波失真

软削波：
x = tanh(x * 1.2) * 0.9
→ 平滑，失真小
```

### 为什么目标振幅是0.6？

**考虑因素**：
- 留一些余量（避免削波）
- 足够大（会议软件能识别）
- 不太大（避免失真）

**经验值**：
- 0.5 - 0.7：最佳范围
- < 0.3：可能太小
- > 0.8：可能失真

### scipy.signal.resample vs 线性插值

**scipy.signal.resample**：
- 使用傅里叶变换
- 保留频率成分
- 音质最好
- 需要安装scipy

**线性插值**：
- 简单快速
- 音质较差（高频衰减）
- 无需额外依赖
