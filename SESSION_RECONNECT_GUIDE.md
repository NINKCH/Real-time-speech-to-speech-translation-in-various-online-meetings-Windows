# 🔄 会话断开自动重连指南

## 问题说明

### 错误信息
```
❌ 会话异常结束: read backend response: rpc error: code = 13 desc = stream terminated by RST_STREAM with error code: NO_ERROR
```

### 原因分析

这是WebSocket连接被服务器主动关闭的错误：

1. **会话超时**：豆包同传API可能有单次会话时长限制
2. **服务器限制**：防止单个连接占用过久
3. **网络波动**：临时的网络问题导致连接中断
4. **资源限制**：服务器资源调度导致的连接重置

**这是正常现象**，不是程序bug！

## 🛠️ 解决方案

### ✅ 已实现：自动重连机制

程序现在会**自动检测连接断开并重连**：

#### 特性

1. **自动检测**：检测到连接错误时自动触发
2. **智能重连**：最多尝试3次重连
3. **无缝继续**：重连成功后继续翻译，音频采集不中断
4. **状态提示**：界面显示重连状态

#### 工作流程

```
连接正常运行
    ↓
检测到连接错误
    ↓
自动触发重连（尝试1/3）
    ├─ 关闭旧连接
    ├─ 等待2秒
    └─ 建立新连接
    ↓
重连成功 → 继续运行 ✅
重连失败 → 继续尝试（2/3, 3/3）
    ↓
所有尝试失败 → 提示用户重启 ❌
```

## 📊 重连配置

可以在 `translator_app.py` 中调整：

```python
# 自动重连配置
self.auto_reconnect = True           # 是否启用自动重连
self.max_reconnect_attempts = 3      # 最大重连次数
self.reconnect_delay = 2             # 重连间隔（秒）
```

### 调整建议

**网络稳定环境**：
```python
self.max_reconnect_attempts = 2
self.reconnect_delay = 1
```

**网络不稳定环境**：
```python
self.max_reconnect_attempts = 5
self.reconnect_delay = 3
```

**禁用自动重连**（不推荐）：
```python
self.auto_reconnect = False
```

**界面显示**：
```
状态: 运行中（已重连）
```

**实际影响**：

## 🔧 手动重启方法

如果自动重连失败：

### 方法1：使用界面按钮（推荐）

```
1. 点击【⏹ 停止翻译】
2. 等待2-3秒
3. 点击【▶ 开始翻译】
```

### 方法2：重启程序

```bash
# 关闭程序窗口
# 然后重新运行
python main.py
```

## 📈 最佳实践

### 1. 长时间使用

如果需要连续使用超过30分钟：

**建议**：定期重启连接
```
每20-30分钟：
1. 停止翻译
2. 等待5秒
3. 重新开始翻译
```

**原因**：
- 避免单次会话过长
- 释放服务器资源
- 保持连接稳定

### 2. 网络不稳定时

**症状**：频繁看到"重连中"

**建议**：
1. 检查网络连接
2. 使用有线网络（而不是WiFi）
3. 关闭其他占用带宽的应用
4. 调高重连次数和延迟

### 3. 会议中使用

**推荐设置**：
```python
self.max_reconnect_attempts = 5  # 多尝试几次
self.reconnect_delay = 2         # 适中的延迟
```

**应急预案**：
- 提前告知会议参与者可能的短暂中断
- 准备好手动重启的步骤
- 考虑使用备用翻译方案
