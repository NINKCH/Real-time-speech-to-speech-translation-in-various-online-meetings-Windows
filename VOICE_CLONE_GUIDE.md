# 🎤 声音克隆功能使用指南

## 📖 功能概述

声音克隆功能可以让翻译后的语音使用您自己的声音，而不是系统默认的机器声音。这样在国际会议、演讲等场合，对方听到的英文翻译将使用您克隆的声音，听起来更加自然和亲切。

## ✨ 功能特性

- ✅ **个性化声音**：使用您自己的声音进行翻译输出
- ✅ **多语言支持**：支持中文、英语、日语、韩语等11种语言
- ✅ **高音质**：24kHz采样率，接近真人音质
- ✅ **低延迟**：实时翻译，声音克隆不影响延迟
- ✅ **持久保存**：Speaker ID自动保存，无需重复配置
- ✅ **内置训练工具**：直接在应用中训练声音（高级功能）

## 🎯 两种使用方式

### 方式一：简单配置（推荐新手）

适合：已经在控制台训练好声音的用户

1. 在控制台训练声音
2. 获取 Speaker ID
3. 在应用中配置使用

### 方式二：内置训练（高级用户）

适合：想要在应用中直接训练的用户

1. 在控制台购买音色获取 Speaker ID
2. 在应用中上传音频训练
3. 训练完成后直接使用

## 🚀 使用步骤

### 步骤1: 在豆包控制台创建声音克隆

1. **访问火山引擎控制台**
   ```
   https://console.volcengine.com/speech/service
   ```

2. **进入语音合成服务**
   - 在左侧菜单选择"语音合成"
   - 找到"音色定制"或"声音克隆"功能

3. **录制声音样本**
   - 按照提示录制您的声音
   - 通常需要20-30分钟的录音素材
   - 建议在安静环境下录制，保证音质清晰

4. **训练声音模型**
   - 上传录音后，系统会自动训练
   - 训练时间：通常需要几小时到1天
   - 您会收到训练完成的通知

5. **获取 Speaker ID**
   - 训练完成后，会生成一个 Speaker ID
   - 格式类似：`voice_12345678` 或 `speaker_xxx`
   - 复制保存此ID

### 步骤2: 在应用中配置

1. **启动应用**
   ```bash
   python main.py
   ```

2. **点击"🎤 声音克隆"按钮**
   - 在主界面上方的紫色按钮

3. **粘贴 Speaker ID**
   - 在对话框中粘贴您的 Speaker ID
   - 点击"保存"

4. **开始翻译**
   - 配置完成后，正常使用翻译功能
   - 翻译的语音将自动使用您的克隆声音

### 步骤2B: 使用内置训练工具（高级）

如果您想直接在应用中训练声音，而不是在控制台：

1. **购买音色**
   - 访问控制台：https://console.volcengine.com/speech/service
   - 购买声音复刻服务
   - 获取 Speaker ID（格式：S_xxxx）

2. **启动应用并打开训练工具**
   ```bash
   python main.py
   ```
   - 点击主界面的 **"🔬 训练工具"** 按钮（橙色）

3. **配置训练参数**
   - **Speaker ID**: 粘贴从控制台获取的 ID
   - **音频文件**: 点击"选择音频"，选择您的录音
     - 支持格式：WAV, MP3, OGG, M4A, AAC, PCM
     - 建议：20秒以上，清晰无噪音，单通道
     - 大小限制：10MB以内
   - **模型类型**: 
     - ICL 2.0（推荐，最新）
     - ICL 1.0
     - DiT 标准版
     - DiT 还原版
   - **音频语种**: 选择您录音使用的语言
   - **降噪选项**: 如果录音有噪音，勾选此项

4. **开始训练**
   - 点击 **"🚀 开始训练"** 按钮
   - 确认配置信息
   - 等待训练完成（通常几分钟）

5. **使用训练好的声音**
   - 训练成功后，返回主界面
   - 点击 **"🎤 配置声音"** 输入 Speaker ID
   - 或者训练工具会自动保存到配置

## 💻 技术实现

### Protobuf协议支持

声音克隆通过在API请求中设置`speaker_id`字段实现：

```python
# 创建StartSession请求
request = TranslateRequest()
request.request.mode = "s2s"
request.request.source_language = "zh"
request.request.target_language = "en"
request.request.speaker_id = "voice_12345678"  # 设置声音克隆ID
```

### 配置持久化

Speaker ID保存在`voice_clone_config.json`文件中：

```json
{
  "speaker_id": "voice_12345678"
}
```

应用启动时自动加载，无需重复配置。

## ⚠️ 注意事项

### 1. 费用说明
- ✅ 基础翻译：按使用时长计费
- ⚠️ 声音克隆：需要额外费用
  - 训练费用：一次性收费
  - 使用费用：可能比标准TTS更贵
- 💡 建议先在控制台查看定价

### 2. 音质要求
- ✅ **录音环境**：安静、无回声
- ✅ **麦克风质量**：建议使用专业麦克风
- ✅ **录音时长**：至少20分钟清晰语音
- ✅ **音频格式**：WAV, 16kHz以上采样率

### 3. 训练时间
- ⏱️ 通常需要 3-12 小时
- 📧 训练完成会收到通知
- 🔄 可以随时重新训练优化

### 4. 隐私安全
- 🔒 声音样本仅用于个人账号
- 🔒 不会被公开或共享
- 🔒 可以随时删除声音模型

## 🔧 故障排除

### 问题1: 找不到声音克隆功能

**解决方案**：
1. 确认您的账号已开通"语音合成"服务
2. 检查是否有"音色定制"权限
3. 联系火山引擎技术支持开通

### 问题2: Speaker ID无效

**检查**：
- ✅ ID格式是否正确（通常以`voice_`或`speaker_`开头）
- ✅ 训练是否完成
- ✅ 是否在正确的账号下使用

**解决**：
- 重新复制ID（避免多余空格）
- 在控制台验证ID状态
- 查看训练日志

### 问题3: 声音效果不理想

**原因**：
- 训练素材质量不高
- 录音时长不够
- 环境噪音太大

**改进**：
1. 重新录制高质量音频
2. 增加录音时长到30分钟以上
3. 使用更好的麦克风
4. 重新训练模型

### 问题4: 翻译使用了默认声音

**检查**：
- ✅ Speaker ID是否正确配置
- ✅ 主界面状态标签是否显示绿色
- ✅ 查看日志是否有"使用声音克隆"提示

**解决**：
```bash
# 查看日志
cat logs/translator.log | grep "声音克隆"

# 应该看到：
# ✨ 使用声音克隆: voice_12345678
```
