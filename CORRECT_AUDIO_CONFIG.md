# ✅ 正确的音频配置指南

## 🎯 重要说明

**Windows的默认输入设备应该保持为真实麦克风，不要改成CABLE Output！**

---

## 📊 三层配置

### 1️⃣ Windows系统层（系统默认）

```
设置 → 系统 → 声音 → 输入
┌────────────────────────────────────┐
│ 输入设备：[您的真实麦克风]  ✅      │
│           (例如：Realtek麦克风)     │
│                                    │
│ ❌ 不要选择：CABLE Output          │
└────────────────────────────────────┘
```

**为什么**：
- 您需要用真实麦克风说话
- 程序从真实麦克风采集声音
- 系统其他功能正常工作

---

### 2️⃣ 程序层（本翻译程序）

```
程序自动配置
┌────────────────────────────────────┐
│ 输入：真实麦克风（自动选择）  ✅   │
│ 输出：CABLE Input（自动配置）✅   │
└────────────────────────────────────┘
```

**无需手动配置**：
- 程序会自动找到真实麦克风
- 程序会自动输出到CABLE Input

---

### 3️⃣ 会议软件层（腾讯会议/Zoom/Teams等）

```
会议软件音频设置
┌────────────────────────────────────┐
│ 麦克风：CABLE Output  ✅           │
│         (VB-Audio Virtual Cable)   │
│                                    │
│ 扬声器：[您的扬声器/耳机]  ✅     │
└────────────────────────────────────┘
```

**关键配置**：
- 麦克风必须选择CABLE Output
- 这样会议对方才能听到翻译

---

## ⚙️ 详细配置步骤

### 步骤1：配置Windows（保持真实麦克风）

1. 右键任务栏音量图标
2. 点击"声音设置"
3. 向下滚动到"输入"
4. **确保选择的是您的真实麦克风**
   - ✅ 例如：Realtek音频
   - ✅ 例如：麦克风阵列
   - ✅ 例如：USB麦克风
   - ❌ 不要选：CABLE Output

**测试**：对着麦克风说话，应该看到音量条跳动

---

### 步骤2：启动翻译程序

```bash
python main.py
```

- ✅ 自动找到真实麦克风

## ❓ 常见问题

### Q1: 为什么不能把Windows默认输入改成CABLE Output？

**A**: 因为CABLE Output是虚拟麦克风，它输出的是程序翻译后的英文。如果设为默认：
- ❌ 程序无法采集您的原声（会形成循环）
- ❌ 系统语音助手等功能异常
- ❌ 其他需要麦克风的应用无法正常工作

### Q2: 那为什么会议软件要选择CABLE Output？

**A**: 因为会议软件可以**单独配置**麦克风，不依赖系统默认设置：
- ✅ 会议软件从CABLE Output读取（翻译后的英文）
- ✅ 程序从真实麦克风读取（原始中文）
- ✅ 两者互不干扰

### Q3: 如何确认配置正确？

**A**: 检查三个地方：

```
✅ Windows输入设备 = 真实麦克风
✅ 程序正在运行，能看到原文译文
✅ 会议软件麦克风 = CABLE Output
```

### Q4: 程序如何知道用哪个麦克风？

**A**: 程序使用的是音频采集库的默认行为：
- 优先使用系统默认输入设备（真实麦克风）
- 或者自动选择第一个可用的真实麦克风
- 不会选择CABLE Output（因为它是虚拟设备）

## 🆘 如果配置错误

### 症状1：程序无法采集声音

**原因**：可能真的把Windows默认改成了CABLE Output

**解决**：
```
Windows声音设置 → 输入 → 改回真实麦克风
```

### 症状2：会议对方听到回声或循环

**原因**：音频形成循环

**解决**：
```
1. 确认Windows默认输入是真实麦克风
2. 确认会议软件选择的是CABLE Output
3. 确认程序没有从CABLE Output采集
```
