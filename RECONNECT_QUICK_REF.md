## 🚀 现在的行为

### 当连接断开时：

```
1. 检测到错误 ⚡
   ↓
2. 自动触发重连 🔄
   ↓
3. 尝试重连（最多3次）
   ↓
4. 成功 → 继续运行 ✅
   失败 → 提示重启 ❌
```

### 您会看到的日志：

**正常重连**：
```
❌ 会话异常结束: read backend response...
⚠️  检测到连接错误，准备自动重连...
ℹ️  尝试重连 (1/3)...
✅ 重连成功！
```

**重连失败**：
```
❌ 会话异常结束...
⚠️  检测到连接错误，准备自动重连...
ℹ️  尝试重连 (1/3)...
❌ 重连失败
ℹ️  尝试重连 (2/3)...
❌ 重连失败
ℹ️  尝试重连 (3/3)...
❌ 所有重连尝试均失败
💡 请手动重启应用
```

## 👤 您需要做什么？

### 大多数情况：什么都不用做！

程序会自动重连，您只需：
- ✅ 继续说话
- ✅ 等待2-5秒
- ✅ 翻译会自动恢复

### 如果重连失败：

```
方法1（推荐）：
1. 点击【⏹ 停止翻译】
2. 点击【▶ 开始翻译】

方法2：
1. 关闭程序
2. 重新运行: python main.py
```

## ⚙️ 配置（可选）

在 `translator_app.py` 第47-50行：

```python
# 自动重连配置
self.auto_reconnect = True           # 启用/禁用
self.max_reconnect_attempts = 3      # 重连次数（默认3次）
self.reconnect_delay = 2             # 间隔秒数（默认2秒）
```

**网络不好时**：改为 `5` 次，`3` 秒间隔
**网络很好时**：改为 `2` 次，`1` 秒间隔

## 📞 常见问题

**Q: 为什么会断开？**
A: 服务器限制单次会话时长，这是正常的。

**Q: 会丢失数据吗？**
A: 重连期间的几秒钟会暂停翻译，但音频采集继续。

**Q: 对方会听到中断吗？**
A: 会有2-5秒的短暂停顿，然后恢复。

**Q: 可以禁用自动重连吗？**
A: 可以，但不推荐。手动重启更麻烦。
