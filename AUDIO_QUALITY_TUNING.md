# 🎵 音频质量调优指南

## 🎯 当前问题：能听到但有杂音

### 问题表现
- ✅ 音频能传输到会议软件
- ✅ 音量足够
- ❌ 有杂音、不清晰

---

## 🔧 优化方案

### 方案1：降低增益和目标振幅（已实施）

**修改**：
- 目标振幅：0.6 → 0.4
- 最大增益：5倍 → 3倍
- 软削波：更温和

**效果**：
- 减少过度放大
- 减少削波失真
- 音质应该改善

**测试**：
```bash
python main.py
# 开始翻译，观察是否还有杂音
```

---

### 方案2：禁用自动增益（如果方案1无效）

如果降低增益后还有杂音，可能TTS音频本身音量就够，不需要增益。

**修改 `modules/virtual_audio.py` 第335-357行**：

```python
# 音频增强处理
# 暂时禁用增益，只使用原始TTS音量
if original_max > 0:
    # 仅软削波，不增益
    audio_float = np.tanh(audio_float * 0.95) * 0.9
    
    final_max = np.max(np.abs(audio_float))
    final_rms = np.sqrt(np.mean(audio_float**2))
    
    print(f"🔊 音频处理（无增益）: 最大振幅 {original_max:.3f}→{final_max:.3f} RMS: {original_rms:.3f}→{final_rms:.3f}")
```

### 方案3：检查重采样质量

**检查是否安装了 scipy**：

```bash
python -c "import scipy; print('✅ scipy已安装')"
```

**如果未安装**：
```bash
pip install scipy
```

**为什么重要？**

| 重采样方法 | 音质 | 说明 |
|-----------|------|------|
| scipy.signal.resample | 高 ✅ | 使用傅里叶变换，保留频率 |
| numpy.interp (线性插值) | 差 ❌ | 简单插值，高频衰减严重 |

**日志中如何判断**：

程序会在重采样时自动选择最佳方法：
- 有scipy：使用 `signal.resample`
- 无scipy：使用 `np.interp`

---

### 方案4：调整软削波参数

如果还是有失真，调整削波强度：

**当前参数**（`modules/virtual_audio.py` 第352行）：
```python
audio_float = np.tanh(audio_float * 0.9) * 0.85
```

**更温和**（减少失真，但音量会更小）：
```python
audio_float = np.tanh(audio_float * 0.7) * 0.8
```

**更激进**（增加音量，但可能失真）：
```python
audio_float = np.tanh(audio_float * 1.1) * 0.9
```

---

### 方案5：调整会议软件设置

**检查会议软件的音频设置**：

1. **自动增益控制（AGC）**
   - 可能会放大小信号的噪音
   - 尝试关闭

2. **降噪功能**
   - 可能误判语音为噪音
   - 尝试关闭

3. **回声消除**
   - 可能影响音质
   - 尝试关闭

4. **音量设置**
   - CABLE Output 的音量是否太大？
   - 尝试降低到50-70%

## 📋 日志分析示例

### 理想日志

```
🔄 重采样: 24000Hz → 48000Hz (18000 → 36000样本)
🔊 音频增强: 最大振幅 0.250→0.400 (增益:1.60x) RMS: 0.080→0.128
🔊 输出TTS音频: 36000字节 → 36000样本 (750ms) → 设备[16]
```

**分析**：
- 原始振幅0.250（正常）
- 增益1.60x（适中，不会失真）
- 最终振幅0.400（合理）
- 应该清晰 ✅

### 问题日志

```
🔄 重采样: 24000Hz → 48000Hz (18000 → 36000样本)
⚠️  增益被限制到3倍（原始振幅太小: 0.080）
🔊 音频增强: 最大振幅 0.080→0.340 (增益:3.00x) RMS: 0.025→0.075
🔊 输出TTS音频: 36000字节 → 36000样本 (750ms) → 设备[16]
```

**分析**：
- 原始振幅0.080（偏小）
- 增益3.00x（达到上限）
- 过度放大可能导致杂音
- **建议**：禁用增益或降低目标振幅
