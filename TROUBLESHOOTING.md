# 故障排除指南

## 音频配置问题

### 正确的三层配置

#### 1. Windows系统层

```
设置 → 系统 → 声音 → 输入
输入设备：[您的真实麦克风]  ✅
```

**重要**: Windows默认输入必须保持为真实麦克风，不要改成CABLE Output！

#### 2. 程序层

程序自动配置：
- 输入：真实麦克风（自动选择）
- 输出：CABLE Input（自动配置）

#### 3. 会议软件层

```
会议软件音频设置
麦克风：CABLE Output  ✅
扬声器：[您的扬声器/耳机]
```

### 完整音频流向

```
[真实麦克风] → [程序采集翻译] → [CABLE Input] → [VB-Cable] → [CABLE Output] → [会议软件]
```

## 常见问题与解决方案

### 问题1: 提示"未找到虚拟音频设备"

**原因**: VB-Audio Virtual Cable 未安装或未重启

**解决**:
1. 下载安装 VB-Audio Virtual Cable
2. 重启电脑
3. 在Windows声音设置中确认有 CABLE Input 和 CABLE Output

### 问题2: 会议软件听不到翻译语音

**检查清单**:
- [ ] 会议软件麦克风选择的是 **CABLE Output**（不是 CABLE Input）
- [ ] 程序运行在 s2s 模式
- [ ] 程序界面显示原文和译文
- [ ] CABLE Output 设备在系统设置中已启用

**解决步骤**:
1. 打开会议软件音频设置
2. 麦克风选择: CABLE Output (VB-Audio Virtual Cable)
3. 测试麦克风，确认音量条有响应

### 问题3: 程序无法采集声音

**原因**: 可能Windows默认输入被改成了CABLE Output

**解决**:
1. Windows设置 → 系统 → 声音 → 输入
2. 改回真实麦克风（如：Realtek麦克风）
3. 测试：对着麦克风说话，应该看到音量条跳动

### 问题4: 连接失败/认证错误

**检查项**:
- [ ] .env 文件存在且格式正确
- [ ] DOUBAO_APP_KEY 已填写
- [ ] DOUBAO_ACCESS_KEY 已填写
- [ ] 已在火山引擎控制台开通同声传译服务
- [ ] 网络连接正常

**解决**:
1. 检查 .env 文件内容：
   ```
   DOUBAO_APP_KEY=your_key（无空格）
   DOUBAO_ACCESS_KEY=your_key（无空格）
   DOUBAO_RESOURCE_ID=volc.service_type.10053
   ```
2. 确认API密钥没有多余空格或换行
3. 测试网络连接

### 问题5: 延迟较高

**可能原因**:
- 网络状况不佳
- 使用了错误的API版本
- 本地网络拥堵

**优化建议**:
1. 使用有线网络连接
2. 关闭其他占用带宽的程序
3. 确认使用豆包AST 2.0 API

### 问题6: 音频断断续续

**可能原因**:
- CPU占用过高
- 其他程序占用音频设备
- 音频块大小设置不当

**解决**:
1. 关闭其他占用CPU的程序
2. 关闭其他使用麦克风的程序
3. 检查 config.py 中的 chunk_duration 设置（默认80ms）

### 问题7: 翻译没有结果

**检查项**:
- [ ] API连接成功（查看日志）
- [ ] 对着麦克风说话了
- [ ] 麦克风音量足够
- [ ] 选择的语言正确

**调试步骤**:
1. 查看 logs/translator.log 日志
2. 确认看到"已连接到服务器"的日志
3. 说话时观察程序界面是否有原文显示

### 问题8: 声音复刻效果不好

**影响因素**:
- 录音环境噪音
- 麦克风质量
- 说话方式

**改进建议**:
1. 使用高质量麦克风
2. 在安静环境使用
3. 清晰自然地说话
4. 保持稳定音量
5. 麦克风距离适中（15-30cm）

## 快速检查清单

### 启动前检查

- [ ] Python环境: 3.12.7
- [ ] 依赖已安装: `pip list` 显示 websockets, PyQt6, sounddevice
- [ ] VB-Cable已安装并重启
- [ ] .env 文件已配置API密钥

### 程序启动后检查

- [ ] 程序显示"虚拟音频输出初始化成功"
- [ ] 对着麦克风说话，程序显示原文
- [ ] 程序显示译文
- [ ] 日志显示"输出TTS音频"

### 会议软件配置检查

- [ ] 麦克风选择了 CABLE Output
- [ ] 麦克风未静音
- [ ] 测试麦克风时有音量指示

## 日志分析

### 查看日志

```bash
# Windows PowerShell
Get-Content logs/translator.log -Wait

# 或直接打开文件
notepad logs/translator.log
```

### 关键日志信息

**正常启动**:
```
豆包实时同声传译器启动
虚拟音频输出初始化成功
找到设备: [X] CABLE Input
```

**正常连接**:
```
已连接到服务器 (Log ID: xxx)
已发送建联请求 (Session ID: xxx)
会话建立成功
```

**正常翻译**:
```
原文: 你好
译文: Hello
输出TTS音频: XXX bytes
```

### 错误日志示例

**API密钥错误**:
```
认证失败: 401 Unauthorized
```

**设备未找到**:
```
未找到虚拟音频设备
```

**网络错误**:
```
WebSocket连接失败: Connection refused
```

## 性能优化

### 降低延迟

1. 使用有线网络
2. 关闭后台程序
3. 使用推荐的 chunk_duration (80ms)

### 提高音质

1. 使用高质量麦克风
2. 减少环境噪音
3. 调整麦克风增益

### 减少CPU占用

1. 关闭不必要的程序
2. 降低GUI更新频率
3. 使用s2t模式（不需要TTS）

## 重置配置

如果配置出现问题，可以：

1. 删除 .env 文件，重新配置API密钥
2. 删除 user_settings.json（用户设置）
3. 检查 config.py 是否被意外修改

## 获取帮助

如果以上方法无法解决问题：

1. 查看 logs/translator.log 日志文件
2. 访问火山引擎控制台提交工单
3. 提供以下信息：
   - 错误信息截图
   - logs/translator.log 日志文件
   - 系统环境信息
